# ViewMarket Handbook

Living documentation for the ViewMarket platform. Every area of the project has its own section here — goals, specs, decisions, and context.

## Sections

### Product & Pages

- [Landing Page](./landing/) — Hero, features, pricing, copy
- [Dashboard](./dashboard/) — Shell, workspace, charts, broker management

### System

- [Architecture](./architecture/) — Stack, API routes, deployment

### Governance

- [Legal](./legal/) — Terms, privacy, compliance
- [Roadmap](./roadmap/) — Phases, sprint goals, backlog

## How to Use

- **Before starting a task:** Read the relevant section for context and goals.
- **After making decisions:** Update the section so it stays current.
- **Adding a new area:** Create a subfolder, add an `index.md`, and link it above.

## Conventions

- Each subfolder has an `index.md` that summarizes the area and links to detail files.
- Use plain Markdown. Keep files scannable — headings, bullet lists, tables.
- Date-stamp decisions where ambiguity might arise later.
